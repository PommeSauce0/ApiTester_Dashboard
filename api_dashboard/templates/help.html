ApiTester

This project uses Python to test your API with predefined mapping.

How to set up

Create your mapping

First, you have to create a JSON file named the same as your API service:
Example: if your request URL is {url}/pokemon/, then you have to name your file pokemon.json

You have to put your mappings in data/mapping/your_service_folder/. Once the JSON file is created, you have to define your API response in it. All following JSON objects are supported by the tester:

  • Number
  • String
  • Boolean
  • Object
  • Array
  • Null

Number type

To define a field of type Number:

{
    "field": {
        "_type": "Number"
    }
}

These properties are available for Number field:

  • _type : a String that defines the type of field (here Number). Returns WrongTypeValue if the type doesn't match
  • _format : (optional) a String that defines the type format of field (can be int or float)
  • _optional : (optional) a Boolean that defines if the field is optional. Returns an error of type MissingFieldError if set to false and the field is missing
  • _nullable : (optional) a Boolean that defines if the field can be null. Returns a WrongValueError if set to false and the field is null

String type

To define a field of type String:

{
    "field": {
        "_type": "String"
    }
}

These properties are available for String field:

  • _type : a String that defines the type of field (here String). Returns WrongTypeValue if the type doesn't match
  • _format : (optional) a String that defines the type format of field (can be int or float)
  • _enums : (optional) an Array containing all enumerations available for this field. Returns a WrongEnumsError if value isn't in this Array
  • _regex : (optional) a String containing a regular expression pattern. Returns RegexError if the pattern doesn't match
  • _optional : (optional) a Boolean that defines if the field is optional. Returns an error of type MissingFieldError if set to false and the field is missing
  • _nullable : (optional) a Boolean that defines if the field can be null. Returns a WrongValueError if set to false and the field is null

Boolean type

To define a field of type Boolean:

{
    "field": {
        "_type": "Boolean"
    }
}

These properties are available for Boolean field:

  • _type : a String that defines the type of field (here Boolean). Returns WrongTypeValue if the type doesn't match
  • _optional : (optional) a Boolean that defines if the field is optional. Returns an error of type MissingFieldError if set to false and the field is missing
  • _nullable : (optional) a Boolean that defines if the field can be null. Returns a WrongValueError if set to false and the field is null

Object type

To define a field of type Object:

{
    "field": {
        "_type": "Object",
        "_properties": {
            "sub_field": {
                "_type": "String"
            }
        }
    }
}

These properties are available for Object field:

  • _type : a String that defines the type of field (here Object). Returns WrongTypeValue if the type doesn't match
  • _properties : an Object containing all properties of this Object. All these fields will be tested and matching errors will be returned
  • _optional : (optional) a Boolean that defines if the field is optional. Returns an error of type MissingFieldError if set to false and the field is missing
  • _nullable : (optional) a Boolean that defines if the field can be null. Returns a WrongValueError if set to false and the field is null

Array type

To define a field of type Array:

{
    "field": {
        "_type": "Array",
        "_values": [
            {
                "_type": "String"
            }
        ]
    }
}

These properties are available for Array field:

  • _type : a String that defines the type of field (here Array). Returns WrongTypeValue if the type doesn't match
  • _values : an Array containing all possible values of this Array. All these values will be tested and matching errors will be returned
  • _minlen : (optional) an Integer that defines the minimal length of this Array
  • _maxlen : (optional) an Integer that defines the maximal length of this Array
  • _optional : (optional) a Boolean that defines if the field is optional. Returns an error of type MissingFieldError if set to false and the field is missing
  • _nullable : (optional) a Boolean that defines if the field can be null. Returns a WrongValueError if set to false and the field is null

You have to start with an unnamed field in your JSON file:

{
    "_type": "Array",
    "_values": [
        {
            "_type": "Object",
            "_properties": {}
        }
    ]
}

Create your service configuration

In the Python file data/service_config.py, you can add your service config in the SERVICE variable.

SERVICE = {
    "pokeapi": {
        "path": "/pokeapi/",
        "url": {
            "localhost": None,
            "snapshot": None,
            "recette": None,
            "production": "https://pokeapi.co/api/v2/",
        },
        "headers": {},
        "uri": "{url}/{api}/",
    }
}

You can then create a new child class in the shortcut.py file

class PokeAPIGlobalTester(GlobalTester):
    def __init__(self, env):
        super().__init__(env, "pokeapi")

Create endpoint's specifications

You can create a YAML file with the same name and location as your mapping file to use some specifications with tests.

extended_path:  # extend path for your API's url
    - '/charizard'
    - '/bulbasaur'

query_specs:  # kwargs for request library (get)
    headers:
        token: 'mon_token'

Each path attribute is optional, even the YAML file itself!

How to test your API

Import the shortcut file and call your child class:

from shortcut import PokeAPIGlobalTester

PokeAPIGlobalTester('production')

You will get a session ID to find your test results in MongoDB.


Dashboard

This project has been created to visualize and analyze the results of the ApiTester project.

Dashboard Guide

The Dashboard allows you to easily view and analyze the results of your API tests. Here's how to navigate and use the Dashboard:

Recent Sessions

The "Recent Sessions" section displays the latest test sessions. Each session is represented with its session ID and an indicator of whether it contains errors or not:

  • - No errors in the session
  • - There are errors in the session

Clicking on a session ID will take you to the detailed results of that session.

Services

The "Services" section shows the success rates of various services based on the recent tests. The success rate is displayed as an icon indicating the overall performance:

  • - 90% or higher success rate
  • - 50% to 89% success rate
  • - 25% to 49% success rate
  • - Below 25% success rate

Hover over the icons to see the exact success rates.

Searching Results

Use the search form on the left to filter test results based on session ID, service, status, environment, or custom MongoDB queries. The results will be displayed on the right.
